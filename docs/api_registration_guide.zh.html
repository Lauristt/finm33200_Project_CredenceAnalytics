先配什么

必须先有

核心运行

  • OPENAI_API_KEY 或 ANTHROPIC_API_KEY：LLM entity extraction、source selection、semantic judge。
  • SEC_USER_AGENT：SEC 不需要 key，但自动化请求应带清晰联系信息。
  • FRED_API_KEY：第一阶段扩 rates、macro、credit spreads、WTI/Brent/gold、major FX 最快。

官方宏观增强

强烈推荐

  • BLS_API_KEY：CPI、PPI、NFP、JOLTS、wages、labor metadata。
  • BEA_API_KEY：GDP、NIPA、PCE、industry、regional accounts。
  • EIA_API_KEY：oil、natural gas、petroleum inventory、energy fundamentals。

市场结构与全球层

按范围接入

  • FINRA_CLIENT_ID / FINRA_CLIENT_SECRET：TRACE / fixed income。
  • OPENFIGI_API_KEY：证券标识映射，key 可选但建议。
  • ECB、BIS、IMF、World Bank、BOE、CFTC：多数可无 key 先接。
工程建议：FRED、BLS、BEA、EIA starter typed adapters 已接入；无 key 层已可跑 SEC、Treasury、GLEIF、CFTC、ECB、BIS、IMF、World Bank、BOE 与 OpenFIGI fallback；FINRA 的授权与条款最重，适合作为第二阶段。

逐个平台注册步骤

FRED / St. Louis Fed 优先注册

写入：FRED_API_KEY

  1. 打开 FRED API Keys。
  2. 登录 FRED account 后 request or view API keys。
  3. 复制 32-character lower-case alpha-numeric key 到 .env 的 FRED_API_KEY。
  4. FRED reader 应使用 fred/series/observations，并在旧新闻验证时传 observation_end={as_of_date}。

BLS Public Data API v2 建议注册

写入：BLS_API_KEY

  1. 打开 BLS registration page。
  2. 填写 email 与 organization name，完成 CAPTCHA。
  3. 收到 registration key 后写入 BLS_API_KEY。
  4. 实现时用 POST https://api.bls.gov/publicAPI/v2/timeseries/data/；payload 至少包含 seriesid、startyear、endyear，有 key 时加 registrationkey。

BLS Public Data API 可公开使用；注册 key 主要用于更完整的 v2 可选参数和更稳的额度。常用 starter series：CPI CUSR0000SA0、core CPI CUSR0000SA0L1E、nonfarm payrolls CES0000000001、unemployment LNS14000000、average hourly earnings CES0500000003、JOLTS openings JTSJOL。

BEA API 建议注册

写入：BEA_API_KEY

  1. 打开 BEA API signup。
  2. 使用有效 email 注册；BEA 会发 36-character UserID。
  3. 把 UserID 写入 BEA_API_KEY。
  4. 实现时先跑 discovery：method=GetDataSetList、method=GetParameterList&DataSetName=NIPA、method=GetParameterValues&DataSetName=NIPA&ParameterName=TableName。
  5. 确认 table/line 后再跑 method=GetData&DataSetName=NIPA&TableName=...&Frequency=...&Year=...&ResultFormat=JSON。

工程上不要把 BEA table id 写死在 verifier 逻辑里；建议单独维护 BEA topic-to-table mapping，并把 table、line、frequency、unit、release/vintage metadata 写进 raw provenance。

EIA Open Data API 商品能源建议注册

写入：EIA_API_KEY

  1. 打开 EIA Open Data registration。
  2. 申请 API key；bulk download 通常不需要 key，但 APIv2 查询需要。
  3. 把 key 写入 EIA_API_KEY。
  4. 实现时使用 APIv2：https://api.eia.gov/v2/{route}/data/?api_key=...。
  5. 先接 route discovery，再接 petroleum spot prices、natural gas、weekly petroleum inventory、electricity / energy balance 等高价值 route。

CFTC COT Public Reporting Environment 通常无需 key

写入：CFTC_BASE_URL；可选 CFTC_APP_TOKEN

  1. 打开 CFTC Commitments of Traders。
  2. 进入 Public Reporting Environment，选择 COT dataset。
  3. 实现 adapter 时通过 PRE / Socrata-style endpoint 过滤 report date、market、commodity group、trader category。

FINRA Query API fixed income 阶段需要

写入：FINRA_CLIENT_ID、FINRA_CLIENT_SECRET

  1. 打开 FINRA Query API 和 Getting Started。
  2. 申请 API Console access。机构账号通常需要 SAA / AA 授权；个人账号可查看可用 credential type。
  3. 在 FINRA Gateway 的 API Console 创建 API Credential，获得 API Client ID。
  4. 根据 FINRA 邮件中的 activation link 设置 API Client Secret；注意 User ID 填 API Client ID，不是登录 FINRA Gateway 的用户名。
  5. 运行时先用 Client ID / Client Secret 通过 FINRA Identify Platform 获取 access token，再调用 https://api.finra.org/data/group/{group}/name/{dataset}。
FINRA fixed-income 数据可能有条款和费用限制。把 FINRA 当作正式 fixed income/TRACE 层，不要把它混同于免费公共宏观 API。

OpenFIGI 建议注册 key

写入：OPENFIGI_API_KEY

  1. 打开 OpenFIGI API documentation。
  2. 注册并登录 OpenFIGI account。
  3. Account page 查看或创建 API key，写入 OPENFIGI_API_KEY。

ECB Data Portal SDMX 无需 key

写入：ECB_BASE_URL=https://data-api.ecb.europa.eu/service

  1. 打开 ECB SDMX web services。
  2. 用 API help 查询 dataflow、key 和 metadata。
  3. 适合 euro-area rates、FX、HICP、banking、payments。

BIS SDMX API 无需 key

写入：BIS_BASE_URL=https://stats.bis.org/api/v2

  1. 打开 BIS Stats API v2 documentation 或 BIS SDMX technical standards。
  2. 无需注册、无需 API key、无需 token；直接通过 HTTPS 请求公开 SDMX RESTful API。
  3. v2 data endpoint 形态是 /data/{context}/{agencyID}/{resourceID}/{version}/{key}，本项目默认用 /data/dataflow/BIS/{dataset}/1.0/{series_key}。
  4. 通过 BIS Data Portal 找到 topic、dataset、series key。
  5. 适合 global banking、debt securities、credit-to-GDP、derivatives、global liquidity。

IMF Data API 无 API key

写入：IMF_BASE_URL=https://api.imf.org/external/sdmx/3.0

  1. 打开 IMF Data APIs。
  2. 公开数据直接用 SDMX 2.1 / 3.0 API 访问，不需要 IMF_API_KEY、token 或 client secret。
  3. 本项目的 public reader 默认使用 SDMX 3.0：/data/dataflow/{agency}/{dataflow}/+/{series_key}。
  4. 当前实现支持显式传 agency/dataflow/key，并内置少量 WEO mapping：GDP growth、inflation、unemployment、current account。
  5. Python 快速试验可以用 sdmx1；项目 adapter 层仍建议保留无依赖的 HTTP client。
  6. Swagger exploration 需要 beta portal account 登录；这只是交互式探索页面，不等于 API key。
  7. 如果访问权限数据，IMF 使用 Azure AD B2C / MSAL 交互式登录拿 bearer token；需要 IMF iData 账号，不是申请 API key。
  8. 适合 WEO、BOP、reserves、fiscal、cross-country macro。

World Bank Indicators API 无需 key

写入：WORLD_BANK_BASE_URL=https://api.worldbank.org/v2

  1. 打开 World Bank Indicators API docs。
  2. 使用 v2 endpoints，例如 /country/all/indicator/SP.POP.TOTL。v1 已下线，不要使用。
  3. Indicators API v2 不需要 API key、token 或任何 authentication。
  4. 常用参数：format=json、date=2015:2023、per_page=500、page=2。
  5. 适合 WDI、IDS、long-run country and development finance indicators。

Bank of England IADB 无需 key

写入：BOE_IADB_BASE_URL

  1. 打开 BOE Database help。
  2. 无需注册、无需 API key、无需 token；使用 URL 参数式 download endpoint。
  3. CSV 基础端点：_iadb-fromshowcolumns.asp?csv.x=yes。
  4. 必填参数：Datefrom、Dateto、SeriesCodes、UsingCodes=Y、CSVF=TN。常用可选参数：VPD=Y、VFD=N。
  5. 本项目默认生成 csv.x=yes&Datefrom=01/Jan/2020&Dateto=now&SeriesCodes={code}&UsingCodes=Y&CSVF=TN&VPD=Y&VFD=N。
  6. CSV/HTML/XML 一次最多 300 个 series codes；Excel 一次最多 250 个。
  7. 适合 UK Bank Rate、GBP rates、FX、monetary and credit statistics。

实现备注

  • credence check 的优先级：官方原始发布 API > 官方聚合 API > licensed/vendor data > web search。
  • FRED 不等于所有宏观 claim 的最终权威源：它是很好的统一入口，但 CPI/PPI/labor 应补 BLS，GDP/PCE/NIPA 应补 BEA，energy commodities 应补 EIA。
  • market data 要标 license/source tier：Stooq/Yahoo/vendor prices 不应和 SEC、BLS、BEA、CFTC、ECB、BIS 这类官方源同权重。
  • SDMX 系接口的难点不是 key：ECB/BIS/IMF 的基础 reader 已接入；下一步主要难在 dataflow、dimension、series key、metadata discovery 和单位/频率归一化。
  • FINRA 是重接入：OAuth2、credential expiry、terms、possible fees 和 dataset entitlement 都要在 adapter 层显式处理。
