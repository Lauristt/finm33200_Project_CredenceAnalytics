What the agent does

Given a financial statement, the pipeline extracts the entity / asset class → classifies the claim type → routes it to a data source → retrieves evidence → produces a verdict, a confidence, and a human-review flag. We evaluate each stage.

Method

Benchmark 1 — real news (113 statements). Real statements from Yahoo Finance across stocks, macro, commodities, FX, indices, rates, crypto and five claim types (numeric, factual, opinion, forecast, causal). Factual statements are labeled true from authoritative sources and each cross-checked against SEC / FRED; a copy of each is perturbed (changed number or date) to make a matched false. Opinion/forecast carry no truth value and test classification only.

Benchmark 2 — SEC-anchored (100 statements). Numbers taken straight from SEC filings, perturbed for false twins — objective but "easy" (the labels come from the same source the agent queries). Comparing the two benchmarks shows how much the easy setup overstates performance. Every claim is traceable in the published per-claim CSVs.

The pipeline never confirmed a single true claim and rated most true claims contradicted — even statements that match SEC to the dollar (NVIDIA Q1 revenue $81.62B = SEC $81,615M; Apple Q2 revenue $111.2B = SEC $111,184M). A trivial "always say false" baseline (80%) beats it, and AUC ≈ 0.5 means it could not tell true from false. The "easy" benchmark (74%) badly overstated this.

Root cause & fix — found via the evaluation

Root cause. The SEC fact extractor kept only the two most-recently-filed values per metric, so recent quarterly figures crowded out the specific fiscal-period value a claim refers to. The figure to verify was simply absent from the evidence → not_found → the verdict logic escalated that to contradicted (a true claim judged false).

Fix A. Surface full-year and quarterly figures, de-duplicated by reporting period (end date + period kind) and tagged for fiscal year/quarter ending YYYY-MM-DD, so the claim's period value is present.

Fix A eliminated the catastrophic behavior — true claims are no longer judged false — and made confidence cleanly separate true from false (AUC 0.50 → 0.83). One residual remains: true claims now top out at partially_supported rather than supported (documented as Fix B).

…but Fix A's benefit is dataset-dependent (the honest catch)

BenchmarkBefore Fix AAfter Fix A
Constructed (claims.json — all plain annual revenue/income)74% · AUC 0.5079% · AUC 0.83
Real news (claims_news — mixed metrics/assets)36% · AUC 0.34535% · AUC 0.338
On the constructed set Fix A looks transformative; on real news it is nearly flat — it reduces true-claim contradictions (30 → 22) but accuracy and AUC barely move, because the A-class it fixes is only ~10% of real failures. The dramatic gain was largely an artifact of a homogeneous, self-built benchmark — the same lesson as 74%-vs-36%, now applied to the fix itself.

The "no evidence" failures are not a missing-key problem. SEC needs no key and FRED is configured — they are (C1) factual claims misclassified as forecast/opinion so the pipeline never queried SEC, and (C2) macro/commodity/index/FX claims never routed to a source (many exist in FRED but the keyword routing didn't map them).

The dominant cause depends on the claim mix. On clean annual revenue/income claims the period bug (A) dominates and Fix A resolves ~19/20. On realistic news, A is only ~10% — the real bottlenecks are retrieval coverage (C, 44%) and segment-metric concept gaps (B, 22%), which Fix A does not touch. A homogeneous, self-built benchmark overstates both the failure rate and the benefit of any single fix.

Does AI help? (per stage)

  • Entity extraction — yes. The LLM layer lifts accuracy 80.5% → 89.4%.
  • Classification — n/a. Rule-based; AI is not used here.
  • Verdict — AI was not the bottleneck. The failure came from a deterministic data-coverage bug, not the LLM judge. Fixing the deterministic extraction unlocked most of the value (AUC 0.50 → 0.83).

Conclusion: AI is necessary but not sufficient — on this task, data coverage and period alignment mattered more than the model.

Failure cases

  1. True claims judged false (pre-fix): NVIDIA $81.62B, Apple $111.2B — both match SEC, both rated contradicted.
  2. Macro routing gap: "U.S. CPI rose 3.8%" returned zero evidence because the claim was not routed to FRED.
  3. Quarterly period confusion: SEC reports overlapping periods (quarter / half-year / annual); matching the wrong one produced false contradictions.

Limitations

  • Verdict accuracy is trustworthy mainly for US equities (SEC, no key); macro/commodity/FX verdicts depend on FRED routing and are partly a coverage gap, not a judgment error.
  • Crypto / exotic credit facts are not reliably checkable by the agent's sources — out of scope.
  • Opinion/forecast have no ground truth; we evaluate only whether they are correctly classified and not fact-checked.
  • News "true" labels were cross-checked against SEC/FRED, but quarterly figures and phrasing remain a known source of label noise.

Bottom line

The agent's front end (entity extraction, classification, routing) works reasonably and AI improves extraction, but its verdict stage shipped with a systematic bug that made it reject true claims — invisible on an easy benchmark, exposed by realistic news data. We located the cause, fixed it (AUC 0.50 → 0.83; true-claim contradictions 19 → 0), and documented the rest. On this task, the win came from fixing the data pipeline, not from the AI model.
