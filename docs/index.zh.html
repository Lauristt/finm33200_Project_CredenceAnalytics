1. 怎么评估的 —— 真实新闻基准

113 条真实语句,从 Yahoo Finance 抓取,覆盖股票/宏观/商品/FX/指数/利率/加密,以及 5 种 claim 类型。事实类标签对 SEC/FRED 核验过,再改数字/日期造出配对假话;观点/预测无真假(用来测分类)。

为什么用真实新闻、而非从数据库拉数据改:这项目的任务是判断真实世界的 statement 准不准。 拿 agent 也会查的同一个数据库里的数字来扰动造题,太简单、会高估表现——所以用真实语句,标签对权威源核验。

3. 错误出在哪 —— 多个原因,没有单一根本原因

我们把真实新闻上 50 条真话的失败逐条归到具体机制:

  1. a. 根本没拿到数据 —— 44%(最大头)。哪些数据:宏观(CPI、非农)、商品(布伦特、铜)、 指数(标普500、道指)、FX(EUR/USD),外加一些被误路由的股票。为什么——而且不是缺 key (SEC 免 key、FRED 已配):
    • 源没实现:系统声明了 25 个数据源(BLS、BEA、ECB、CFTC…),但只有 ~12 个真有取数函数, 其余只是菜单"卡片"、没代码 → 非股票类几乎只剩 FRED 能用。
    • 路由太窄:FRED 其实有 CPI/非农/WTI/标普/EUR-USD,但关键词表只认 "cpi"/"inflation"/"gdp", 认不出 "consumer prices""payrolls""Brent" → 压根没去查。
    所以多数情况是"数据在一个被覆盖的源里、但没被接通/路由",而不是"数据本来就没有"(只有加密货币是真没源)。
  2. b. 把时间期间比错了 —— 10%(我们发现并修的 Fix A)。SEC 同时有季度/半年/年度等重叠期间; 它拿真话去比错了期间 → 判成矛盾。Fix A(按期间标注列值)修好了这一类——但它只占真实失败的 ~10%, 所以真实新闻准确率仍 ≈35%:瓶颈在别处(数据覆盖)。
  3. c. 数据找到了也不敢判"真" —— 系统性。即便数字对上,判定也停在 partially_supported, 几乎到不了 supported(真话只确认 1/50)。判定逻辑过于保守。
  4. d. 分类错 + 概念缺失。事实被标成 forecast/opinion → 不去核查(9 条); 细分指标(iPhone 营收、AWS)不在映射概念里 → 取不到(11 条);部分实体(纳指、GBP/JPY)没被提取(3 条)。

4. AI 有没有用?(分阶段)

  • 实体提取——有用(80.5% → 89.4%)。
  • 分类——不适用(纯规则,不用 AI)。
  • 判定——AI 不是瓶颈;失败在数据覆盖和路由,不在模型。AI 必要但不充分。

5. 局限

  • 判定准确率主要对美股(SEC)可信;非股票覆盖受限于"源没实现 + 路由太窄",不是缺 key。
  • 系统宣称 25 个数据源,但只有 ~12 个真正实现——大部分"覆盖"是有菜单没后端。
  • 观点/预测无客观真假(只测分类 + 路由);加密/冷门资产超出范围。

6. 结论

前端(实体提取、分类、路由)基本能用,AI 还提升了提取。判定环节在真实声明上不可靠,但没有单一根本原因: ~44% 是"没拿到数据"(源没实现或没路由——不是缺 key),~22% 是概念缺失,只有 ~10% 是我们修的期间 bug, 还有一种系统性的保守让它不敢确认真话。可靠性瓶颈在数据覆盖,不是某个单点 bug。
