这个 agent 在做什么

给定一句金融声明,流程为:提取实体/资产类别 → 判定 claim 类型 → 路由到数据源 → 检索证据 → 给出判定(verdict)、置信度、人工复核标记。我们对每个阶段分别评估。

方法

Benchmark 1 — 真实新闻(113 条)。从 Yahoo Finance 抓取真实语句,覆盖 股票、宏观、商品、FX、指数、利率、加密货币,以及 5 种 claim 类型(numeric、 factual、opinion、forecast、causal)。事实类语句默认标为 true(来自权威来源), 且每条都对 SEC / FRED 核验过;再各复制一份、改数字或日期,造出配对的 false。观点/预测无真假,用来测分类。

Benchmark 2 — SEC 锚定(100 条)。数字直接取自 SEC 财报、扰动造假——标签客观, 但“太简单”(标签来自 agent 也会去查的同一个数据源)。两个 benchmark 对比,能看出简单设置把 表现夸大了多少。每条结果都可在公开的逐条 CSV 里追溯。

该 pipeline 一条真话都没确认,还把大多数真话判成 contradicted—— 连和 SEC 一字不差的语句也是(NVIDIA Q1 营收 $81.62B = SEC $81,615M;Apple Q2 营收 $111.2B = SEC $111,184M)。一个“永远说假”的笨基线(80%)都能赢它;AUC ≈ 0.5 说明它根本分不出真假。 而“简单”benchmark(74%)严重高估了它。

根因与修复 —— 由评估定位

根因。SEC 取数器每个指标只保留最近申报的 2 个值,于是近期季度数把 声明真正引用的那个财务期间值挤掉了。要核对的数字根本不在证据里 → not_found → 判定逻辑又把它升级成 contradicted(把真话判成假)。

Fix A。同时列出年度与季度数值,按报告期间(期末日 + 期间类型)去重,并标注 for fiscal year/quarter ending YYYY-MM-DD,确保声明对应期间的值出现在证据里。

Fix A 消除了最严重的问题——真话不再被判成假——并让置信度能干净地区分真假(AUC 0.50 → 0.83)。 还剩一个残留:真话目前停在 partially_supported 而非 supported(记为 Fix B)。

失败归因 —— 多个原因,不止一个

我们把修复前的每条失败(50 条真话全部未确认)通过查看 pipeline 内部信号,逐条归到具体机制:

主要失败原因取决于题目构成。在干净的年度营收/净利润题上,期间 bug(A)占主导, Fix A 修掉 ~19/20;但在真实新闻上 A 只占 ~10%——真正的瓶颈是检索覆盖(C,44%)和细分指标概念缺失 (B,22%),Fix A 都不碰。同质化的自造题会同时高估失败率和单个修复的效果。

"没拿到数据"基本不是缺 key 的问题。SEC 免 key、FRED 也配了——真正原因是 (C1)事实被误判成预测/观点,系统就没去查 SEC;(C2)宏观/商品/指数/FX 类没被路由到数据源 (很多其实 FRED 里有,只是关键词没映射过去)。

AI 有没有用?(分阶段)

  • 实体提取——有用。LLM 层把准确率从 80.5% 提到 89.4%。
  • 分类——不适用。纯规则,这里不用 AI。
  • 判定——AI 不是瓶颈。失败源于一个确定性的数据覆盖 bug,而非 LLM judge; 修好确定性取数后,价值才释放出来(AUC 0.50 → 0.83)。

结论:AI 是必要但不充分的——在这个任务上,数据覆盖与期间对齐比模型本身更关键。

失败案例

  1. 真话被判成假(修复前):NVIDIA $81.62B、Apple $111.2B——都与 SEC 吻合,却都被判 contradicted。
  2. 宏观路由漏洞:“美国 CPI 上涨 3.8%”检索到 0 条证据,因为没被路由到 FRED。
  3. 季度期间混淆:SEC 含重叠期间(季度/半年/年度);比错期间就产生了假矛盾。

局限

  • 判定准确率主要对美股可信(SEC,免 key);宏观/商品/FX 依赖 FRED 路由, 部分是覆盖盲点而非判断错误。
  • 加密货币/冷门信用资产的事实,agent 的数据源无法可靠核验——超出范围。
  • 观点/预测没有客观真假;只评估它们是否被正确分类并不去核查。
  • 新闻“真”标签已对 SEC/FRED 核验,但季度数与措辞仍是已知的标签噪声来源。

结论

该 agent 的前端(实体提取、分类、路由)基本能用,AI 还提升了提取,但其 判定环节出厂带着一个系统性 bug,会把真话拒判为假——在简单 benchmark 上看不出来, 被真实新闻数据暴露。我们定位了根因、修复了它(AUC 0.50 → 0.83;真话冤判 19 → 0),并记录了余下工作。 在这个任务上,真正的提升来自修数据管线,而不是 AI 模型本身。
